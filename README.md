# Реализация утилиты `grep`

## Что может?
Поддерживает следующие флаги (включая длинные версии GNU):
- `-e PATTERN` - задаёт шаблон для поиска (можно использовать несколько `-e`)
- `-i` (`--ignore-case`) - игнорировать регистр символов
- `-v` (`--invert-match`) - инвертировать поиск (выводить строки, не совпадающие с шаблоном)
- `-c` (`--count`) - выводить только количество совпадающих строк
- `-l` (`--files-with-matches`) - выводить только имена файлов, содержащих совпадения
- `-n` (`--line-number`) - выводить номера строк
- Мультифлаговость: допустимы любые комбинации флагов (например, `-v -c`, `-n -l`, `-i -v -c` и т.д.)
- Поддерживает работу с несколькими файлами (если указано больше одного, перед строкой выводится имя файла) и со стандартным вводом (`-` или отсутствие файлов)
- Использует регулярные выражения (библиотека `regex.h`) для поиска
- Выводит сообщения об ошибках (идентичные оригинальной утилите `grep`) в `stderr`
- Имеется набор тестов с автоматическим сравнением результатов с системной утилитой `grep`

### Что нужно?
- Наличие GCC компилятора 

### Сборка и тестирование
- `make` - компиляция исполняемого файла `grep`
- `make test` - запуск тестов (сравнение с системным `grep`)
- `make clean` - удаление всех созданных файлов при запуске команд выше
